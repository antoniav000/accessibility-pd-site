Module 3: Accessible PDFs

What is a Screen Reader?

A screen reader is a type of assistive technology that converts digital text and interface elements into audio or braille output. It is primarily used by people who are blind or have low vision, but it can also help users with learning disabilities, injuries, or reading difficulties.

The purpose of a screen reader is to allow users to navigate websites, apps, documents, and software without relying on vision. It reads aloud:

  • Text
  • Buttons
  • Links
  • Headings
  • Form fields
  • Image descriptions (alt text)

Some common screen readers are JAWS, NVDA, VoiceOver (for MacOS), and TalkBack.

How do screen readers work?

Linear Navigation

Screen readers usually read content in a linear order, from top to bottom and left to right in the sturcture of the page.

Reading Structure

Screen readers use HTML structure and accessibility tags to understand content.

Important structural elements include:

  • Headings (H1, H2, H3)
  • Landmarks (navigation, main content, footer)
  • Lists
  • Tables
  • Form labels
  • Buttons and links

For example:

  • An H1 tells the user the main topic of the page
  • H2s organize sections

Without structure, a page might seem like one long block of text.

What are the best practices for PDF accessibility?

Proper Headings

Use headings in order:

  • H1 for main page title
  • H2 for section headings
  • H3 for subsections

Correct structure helps users navigate quickly. It is also helpful to avoid skipping levels (like H1 directly to H4).

Alt Text Writing

Alt text describes image for users who cannot see them.

  • Good alt text: “Students using a laptop with a screen reader enabled”
  • Bad alt text: “image123.jpg”

Decorative images should usually be marked as decorative so that they are skipped by screen readers.

Link Clarity

Links should always describe their destination or purpose. For example:

  • Good: "Download the accessibility checklist"
  • Bad: "Click here"

Making PDFs and Files Accessible

PDFs can become inaccessible when they are created from scanned images instead of real text, lack headings and document structure, have missing alt text for images, use incorrect reading order, or contain unlabeled tables and links.

Start with the Source File Principle

The easiest way to create an accessible PDF is to make the original document accessible first.

Examples of source files

  • Microsoft Word documents
  • Google Docs files
  • Microsoft PowerPoint slides

If you can create an original document for accessibility, when exported correctly, any accessibility features carry over into the PDF automatically. It is b est practice to make the original file accessible, export to a PDF format, run an accessibility check, and fix any remaining issues.

Steps for Accessibility

  • Check structure (make sure document has real headings, lists created with list tools, and proper table headers)
  • Add alt text (describe meaningful images)
  • Export properly (enable accessibility/tagging options when saving/exporting as PDF)
  • Run an accessibility checker (Adobe Acrobat Pro Accessibility Checker, Microsoft Word Accessibility Checker)

Fixing Existing PDFs (Using Adobe Acrobat Pro)

Main tools in Acrobat:

  • Accessibility Checker
  • Reading Order tool
  • Tags panel
  • Set alternate text
  • Autotag Document

Here is the official Adobe Acrobat Accessibility Guide.

Making PDFs Accessible in Adobe Acrobat Pro

Alternative to Adobe Acrobat Pro

Adobe Acrobat Pro has the limitation of being a subscription only software, so many accessibility features are not available with the free version. An alternative way to make PDFs more accessible would be to use a separate OCR tool to create recognizable text and edit this text in Word to make it more accessible. OCR (Optical Character Recognition) converts scanned image text into selectable, readable text. Without OCR, screen readers cannot read the document properly and text cannot be searched or highlighted.

Here are some alternative OCR methods to Adobe Acrobat Pro:

  • Tesseract
  • Umi-OCR

After using OCR to create readable and editable text, edit this in Word for accessibility.

Key Takeaway

Accessible PDFs are much easier to create when accessibility is built into the original document first. OCR, tagging, headings, alt text, and proper reading order are all essential for ensuring screen reader compatibility.
